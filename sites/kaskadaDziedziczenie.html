Kaskada i dziedziczenie

CSS działa kaskadowo — oznacza to, że kilka stylów może dotyczyć tego samego elementu, a przeglądarka musi zdecydować który wygra.

1. Kaskada – kolejność ma znaczenie

p {
    color: red;
}

p {
    color: blue;
}

Ten tekst jest NIEBIESKI, bo druga reguła nadpisuje pierwszą.

2. Specyficzność – kto ma większą siłę

p {
    color: red;
}

.klasa {
    color: green;
}

#id {
    color: blue;
}

Zwykły p (czerwony)

Klasa (zielony)

ID (niebieski – najsilniejszy)

3. !important – wymuszenie stylu

p {
    color: red !important;
}

Ten tekst byłby nadpisany, ale !important wymusza kolor.

4. Dziedziczenie – elementy przejmują styl

div {
    color: blue;
}

Ten tekst dziedziczy kolor z diva

Każdy element wewnątrz też jest niebieski

5. Co NIE jest dziedziczone

Nie wszystkie style się dziedziczą — np. border, margin czy background. 

div {
    border: 2px solid red;
}

Ten tekst NIE ma ramki

6. inherit – wymuszenie dziedziczenia

p {
    color: inherit;
}

Ten tekst wymusza dziedziczenie koloru
