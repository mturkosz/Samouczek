Wprowadzenie

HTML (HyperText Markup Language) to nie język programowania, a hipertekstowy język znaczników używany do tworzenia stron internetowych. Służy do budowania struktury strony, czyli określa co znajduje się na stronie, np. nagłówki, akapity, obrazy, linki czy formularze. HTML nie odpowiada za wygląd strony (mimo, że kiedyś to robił) - tylko za jej „szkielet”. Jego specyfikację można znaleźć na html.spec.whatwg.org. Jego parserem (analizatorem składni) jest validator.w3.org.

CSS (Cascading Style Sheets) to język odpowiedzialny za wygląd strony internetowej. Dzięki CSS możemy zmieniać kolory, czcionki, układ elementów, odstępy oraz tworzyć animacje i responsywne strony, które działają dobrze na telefonach i komputerach. Wyróżniamy trzy typy stylów: zewnętrzny (osobny plik, w HTML dołączony arkusz), wewnętrzny (umieszczony w <head>) i lokalny (umieszczony wewnątrz tekstu). Parserem CSS jest np. jigsaw.w3.org.

HTML i CSS współpracują ze sobą: HTML tworzy strukturę, a CSS nadaje jej styl i wygląd. Bez CSS strona byłaby prosta i surowa, a bez HTML nie byłoby na czym stosować stylów. Dlatego więc i HTML, i CSS jest bardzo istotny dla strony.
